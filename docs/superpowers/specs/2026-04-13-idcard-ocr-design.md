# 身份证 OCR 识别系统设计

## 概述

基于 Python 的命令行身份证识别脚本，使用 OpenCV 进行图像预处理和区域裁剪，使用 PaddleOCR 进行分区文字识别，输出结构化的身份证正面信息。

## 需求

- **使用方式**：命令行脚本，传入图片路径
- **识别内容**：身份证正面 6 个文字字段（姓名、性别、民族、出生日期、住址、身份证号）
- **OCR 引擎**：PaddleOCR（pip 安装，离线运行）
- **输出方式**：终端打印结构化结果

## 技术选型

| 组件 | 选择 | 用途 |
|------|------|------|
| OCR 引擎 | PaddleOCR | 中文文字识别 |
| 图像处理 | OpenCV | 预处理、轮廓检测、透视变换、区域裁剪 |
| 数组运算 | NumPy | 坐标计算 |

## 架构

```
输入图片 → [1. 图像预处理] → [2. 身份证定位与矫正] → [3. 区域裁剪] → [4. 分区OCR + 结果输出]
```

### 阶段 1：图像预处理

- 读取原图 → 转灰度 → 高斯模糊降噪 → Canny 边缘检测
- 目的：找到身份证的外轮廓边缘

### 阶段 2：身份证定位与矫正

- 对边缘图做轮廓检测，找到最大的四边形轮廓（身份证边框）
- 对四个顶点排序（左上、右上、右下、左下）
- 用透视变换（`getPerspectiveTransform`）将身份证矫正为标准矩形
- 矫正后缩放到标准尺寸 **640x400**（身份证比例 85.6mm x 54mm）
- 容错：如果找不到四边形轮廓（图片已是裁好的身份证），跳过矫正，直接缩放

### 阶段 3：区域裁剪

身份证正面布局固定，按比例裁剪各字段区域：

```
┌─────────────────────────────────────────────┐
│                                        ┌───┐│
│  [姓名区域]                            │   ││
│                                        │照 ││
│  [性别区域]      [民族区域]            │   ││
│                                        │片 ││
│  [出生日期区域]                        │   ││
│                                        │区 ││
│  [住址区域]                            │域 ││
│  (可能占两行)                          │   ││
│                                        └───┘│
│  [身份证号区域]                              │
└─────────────────────────────────────────────┘
```

每个区域用 `(x, y, w, h)` 的比例系数定义，裁剪时乘以实际图片尺寸（640x400）。

各区域大致比例范围（基于标准身份证布局）：

| 字段 | x 比例 | y 比例 | 宽比例 | 高比例 |
|------|--------|--------|--------|--------|
| 姓名 | 0.15 | 0.08 | 0.35 | 0.12 |
| 性别 | 0.15 | 0.22 | 0.10 | 0.10 |
| 民族 | 0.35 | 0.22 | 0.15 | 0.10 |
| 出生日期 | 0.15 | 0.35 | 0.45 | 0.10 |
| 住址 | 0.15 | 0.48 | 0.45 | 0.20 |
| 身份证号 | 0.10 | 0.78 | 0.80 | 0.14 |

具体数值在实现时根据测试图片微调。

### 阶段 4：分区 OCR + 结果输出

- 对每个裁剪区域单独调用 PaddleOCR 识别
- 身份证号区域额外做一次二值化增强（数字较小，提高识别率）
- 对 OCR 结果做简单清洗（去除多余空格、特殊字符）

## 输出格式

```
========== 身份证识别结果 ==========
姓名：朱礼涛
性别：男
民族：汉
出生日期：1987年2月8日
住址：湖北省蕲春县赤东镇田围村4组
身份证号：421126198702080813
================================
```
